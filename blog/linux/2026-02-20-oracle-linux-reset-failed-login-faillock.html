Oracle Linux 8: Reset Failed Login Attempts Using Faillock

February 20, 2026 3 min read

Here's a quick guide on how to reset the failed login attempts counter for a user, Miki, in Oracle Linux. I'm using these exact steps on my Oracle Linux 8.10 system.

Check Failed Login Attempts

Use the faillock tool to check how many unsuccessful login attempts a user has:

sudo faillock --user Miki

Example output:

Miki:
When                Type  Source                                           Valid
2026-02-19 12:58:36 RHOST 10.10.10.1                                           V
2026-02-19 12:58:41 RHOST 10.10.10.1                                           V
2026-02-19 12:58:46 RHOST 10.10.10.1                                           V
2026-02-19 12:58:52 RHOST 10.10.10.1                                           V
2026-02-19 12:59:07 RHOST 10.10.10.1                                           V

💡 What does the "V" mean? The V in the Valid column stands for "Valid", meaning this failed login attempt is still being counted against the user. Once the lockout period expires or the counter is reset, these entries will no longer appear.

Check Lockout Configuration

You can verify how many login attempts are allowed before lockout by viewing the faillock configuration:

cat /etc/security/faillock.conf | grep deny

This will show you the deny parameter which specifies the number of failed attempts before the account is locked.

Reset the Failed Login Counter

To reset the failed login count and unlock the user:

sudo faillock --user Miki --reset

Verify the Reset

Run the first command again to confirm the counter has been cleared:

sudo faillock --user Miki

You should now see no entries:

Miki:
When                Type  Source                                           Valid

Next Steps

That's all there is to it. You can now ask the user to try logging in again. If they've forgotten their password, you can reset it using:

sudo passwd Miki

Quick Reference

Task Command
Check failed attempts sudo faillock --user <username>
Reset failed counter sudo faillock --user <username> --reset
Check lockout config cat /etc/security/faillock.conf
Reset user password sudo passwd <username>
