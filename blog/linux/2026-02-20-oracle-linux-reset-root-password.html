Oracle Linux 8: How to Reset Root Password

February 20, 2026 5 min read

Let's cut to the chase for this topic. Here's the environment:

  • Objective: Reset root password
  • OS: Oracle Linux 8.10

To reset the root password, you need to enter single user mode. This means rebooting the VM and accessing the GRUB bootloader.

⚠️ Important: To access the GRUB bootloader, you must have console access to the server. This won't work over SSH.

Step 1: Shut Down the VM

I'm using vCenter for this. Select your VM, click on Power, then Shut Down Guest OS. Wait for your VM to fully power off.

Step 2: Interrupt the Boot Process

You need to be quick for this step because we have to interrupt the boot process before the OS loads.

Power on the VM and open the console. When you see the kernel selection screen, quickly press the e key.

💡 Note: You can see I have 6 versions of kernel on this VM. At first, I was a bit concerned about picking the wrong kernel, but the OS will highlight your current kernel during the boot process. So you can safely press e on the highlighted entry. The picture above shows that I'm entering the GRUB bootloader for kernel Oracle Linux Server 8 (5.4.17-2136.5.1.el8uek.x86_64).

Step 3: Edit the Boot Parameters

From the GRUB bootloader screen, navigate to the line that starts with linux. You can use the arrow keys to move down.

Go to the end of the line and type rd.break.

Take note that it says to press Ctrl+X to start. Press Ctrl+X to boot into single user mode.

Step 4: Reset the Root Password

Now you're in single user mode. Run the following commands to change the root password:

# Remount /sysroot with read-write permissions
mount -o remount,rw /sysroot

# Change root directory to /sysroot
chroot /sysroot

# Change the root password
passwd

# Create the autorelabel file for SELinux
touch /.autorelabel

# Exit chroot
exit

What Do These Commands Do?

mount -o remount,rw /sysroot

This command remounts the /sysroot filesystem with read-write permissions. By default, single user mode mounts the filesystem as read-only to prevent accidental changes. We need write access to modify the password file.

chroot /sysroot

The chroot command changes the apparent root directory to /sysroot. This makes the system treat /sysroot as if it were the actual root (/) directory, allowing us to run commands like passwd against the real system files rather than the rescue environment.

touch /.autorelabel

This creates an empty file that tells SELinux to relabel all files on the next boot. Since we modified the password file outside of the normal SELinux context, the security labels are now incorrect. Without relabeling, SELinux may block access to the modified files and you could be locked out of the system.

Step 5: Verify

Once the VM is fully booted, verify that you can log in as root using your new password.

Wrapping Up

Resetting the root password in Oracle Linux 8 is straightforward once you know the steps. The key things to remember are: you need console access, add rd.break to the kernel boot parameters, remount the filesystem as read-write, and don't forget to create the /.autorelabel file for SELinux.
