Fixing the "Collecting but Not Receiving Data" Warning in vROps NSX ALB Adapter

January 15, 2026 4 min read

Ran into an odd one recently with VMware vRealize Operations 8.10 and the NSX Advanced Load Balancer adapter. The adapter was stuck in a Warning state specifically, "Collecting but Not Receiving Data." Everything looked fine, but vROps wasn't happy. Here's what happened and how I fixed it.

The Problem

Administrative alerts shows the NSX ALB adapter status showed a yellow warning icon. The message said it was collecting but not receiving data. Not ideal when you're trying to monitor your load balancers.

The frustrating part? No obvious errors. The adapter was configured correctly, credentials were valid, and the NSX ALB controller was reachable. Yet the warning persisted.

What I Tried First

My first instinct was to validate the connection. In vROps, I went to the adapter instance and hit VALIDATE CONNECTION multiple times. The first time, it prompted me to accept new certificate. Each time: success. No errors, no complaints. The adapter could clearly talk to the NSX ALB controller.

I waited. Sometimes these things just need time to sync up. But after several collection cycles, the warning was still there. The adapter stubbornly refused to show actual data coming in.

I double-checked the usual suspects:

  • Credentials: Correct
  • Controller URL: Correct
  • Network connectivity: Fine
  • Permissions on the NSX ALB side: All good

Everything checked out, but the problem remained.

The Fix: Rebooting the vROps Cluster

After exhausting the obvious options, I decided to restart the vROps cluster. Sometimes that's just what it takes.

In the vROps admin interface, I initiated a cluster restart and waited for all nodes to come back online. It takes a few minutes, so grab a coffee.

Back to Normal

Once the cluster was fully back up and the services had stabilized, I checked the adapter instance again.

After the reboot, the adapter status changed to OK. Green icon, collecting data, receiving data. Everything working as expected. The NSX ALB metrics started flowing into vROps without any further intervention.

Why Did This Work?

Honestly? I'm not 100% sure. My best guess is that something got stuck in the collector service. Maybe a hung process or a stale connection that wasn't releasing properly. The reboot cleared whatever was blocking the data flow.

This isn't the most satisfying answer, but in the VMware world, sometimes a restart fixes things that have no clear root cause. The important thing is documenting it so future-me (or you) doesn't spend hours chasing ghosts.

Key Takeaways

  • "Collecting but Not Receiving Data" doesn't always mean a configuration problem
  • Successful connection tests don't guarantee the adapter is fully functional
  • When in doubt, reboot the cluster - it's not elegant, but it works
  • Document these fixes - you'll forget, and you'll hit this again in six months

If you're running vROps 8.10 with the NSX ALB adapter and see this warning, try the connection validation first. But if that comes back clean and the warning persists, a cluster reboot might be your answer.
