AIX Filesystem Management: How to Check and Resize Mount Points

February 17, 2026 5 min read

This note is one I always refer to when working with AIX, and I hope anyone still managing AIX systems finds it useful. Here's how I usually handle increasing the size of mount points.

Checking Filesystem Size

df -g

View Logical Volume Details

lslv <logical_volume_name>

View Volume Group Information

# View volume group details
lsvg <volume_group_name>

# View physical volumes in a volume group
lsvg -p <volume_group_name>

Adding or Reducing Filesystem Size

Basic Syntax

chfs -a size=+/-<amount><unit> <filesystem_path>

Examples

# Increase /opt by 4GB
chfs -a size=+4G /opt

# Increase /usr by 6GB
chfs -a size=+6G /usr

# Reduce /tmp by 500MB
chfs -a size=-500M /tmp

Units:

  • G = Gigabytes
  • M = Megabytes
  • No unit = 512-byte blocks

Reduce Filesystem Size

chfs -a size=-<amount><unit> <filesystem_path>

⚠️ Warning: Be very careful when reducing filesystem size! Always ensure the filesystem has enough free space before shrinking it.

Wrapping Up

These are the commands I use most frequently when managing filesystems in AIX. The chfs command makes it straightforward to resize filesystems on the fly, and the various lsvg and lslv options help you understand your storage layout before making changes.
