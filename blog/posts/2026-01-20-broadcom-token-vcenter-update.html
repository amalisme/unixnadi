How to Use Broadcom Token to Update vCenter Server via VAMI

January 20, 2026 3 min read

I was planning to update my vCenter to 8.0 Update 3g next week to resolve VMSA-2025. While running through my pre-check activities, noticed that the VAMI couldn't connect to Broadcom's update repository. The check for updates was failing silently. No available updates showing up at all.

Since I'd just gone through a similar exercise with ESXi Lifecycle Manager, I figured the fix would be the same: use my Broadcom token. Turns out I was right. The process for vCenter is slightly different (you configure it in VAMI settings rather than Lifecycle Manager), but the concept is identical.

Here's how to get vCenter updates working again using your Broadcom token.

Pre-Requisites

Get Your Broadcom Token

If you haven't already, you'll need to generate your unique download token from the Broadcom Support Portal. Follow the steps in Broadcom KB 390098 to create one.

Good news: if you already generated a token for ESXi updates, you can use the same token here. It's tied to your Broadcom account, not to a specific product.

VAMI Access

You'll need root access to the vCenter VAMI interface. Make sure you have the root password handy before starting.

Configure the Update URL in VAMI

Step 1: Log into VAMI

Open your browser and navigate to https://<your-vcenter-ip-or-fqdn>:5480. Log in using the root account.

Step 2: Open Update Settings

Click on the Update tab in the left sidebar, then click Settings in the top right area of the Update page.

Step 3: Change the Repository URL

By default, VAMI uses VMware's old update URL which no longer works after the Broadcom acquisition. You need to switch to a specified URL with your token.

Select Specified instead of Default, then enter the new Broadcom URL. The URL format depends on your current vCenter version. For vCenter 8.0 Update 3b, the change looks like this:

Default URL (no longer works):

https://vapp-updates.vmware.com/vai-catalog/valm/vmw/8d167796-34d5-4899-be0a-6daade4005a3/8.0.3.00100.latest/

New Broadcom URL (replace with your token):

https://dl.broadcom.com/<your-download-token>/PROD/COMP/VCENTER/vmw/8d167796-34d5-4899-be0a-6daade4005a3/8.0.3.00100/

üí° Note: The version number in the URL path (8.0.3.00100 in this example) corresponds to your current vCenter version. Check Broadcom KB 390120 for the complete list of version-specific URLs.

Step 4: Save and Verify

Click Save to apply the new URL. Go back to the main Update tab and click Check Updates. If everything is configured correctly, you should now see available vCenter updates, including 8.0 Update 3g.

What's Next

Once you can see the target version in VAMI, you're ready to proceed with the actual update. If you need a walkthrough of the update process itself, check out my guide on updating vCenter 8.0 Update 3b to 3g via URL.

Wrapping Up

The Broadcom repository changes affect both ESXi and vCenter updates, so you'll likely need to configure tokens in both places. The good news is that once it's set up, it works reliably.You just need to remember to do it again if you rebuild or deploy a new vCenter.

If you're hitting other issues with vCenter updates failing, the error messages in VAMI are usually helpful. Most problems come down to either connectivity (firewall blocking the Broadcom URLs) or an incorrectly formatted token URL.
