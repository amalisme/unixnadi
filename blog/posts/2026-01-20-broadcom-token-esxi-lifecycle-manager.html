How to Use Broadcom Token to Download ESXi Updates in vCenter Lifecycle Manager

January 20, 2026 4 min read

I ran into an issue while trying to update my ESXi hosts using vCenter Lifecycle Manager last year (to resolve VMSA-2025) and the latest releases weren't showing up. Turns out, after the Broadcom acquisition, the old VMware repository URLs no longer work. You now need to generate a unique token from the Broadcom portal and use their new depot URLs.

This is the error message you can see from the vCenter Task Console

I found the Broadcom KB article explaining the process, but I'll be honest‚ÄîI was a bit hesitant to just dive in without understanding exactly what I was doing, this is the on-prem for one of our core customer, and I have several questions as well. Long story short, after working through it, I can confirm it's straightforward once you know the steps.

Here's how to get your ESXi updates flowing again.

Pre-Requisites

Generate Your Broadcom Token

First, you need to obtain your unique download token from the Broadcom Support Portal. This token is tied to your account and is used to authenticate your access to the software repository.

Follow the steps in Broadcom KB 390098 to generate your token. Once you have it, keep it handy. You'll need it for the next step.

Build Your Repository URLs

With your token in hand, you need to construct the new depot URLs. Replace <Download Token> in each URL below with your actual token:

https://dl.broadcom.com/<Download Token>/PROD/COMP/ESX_HOST/main/vmw-depot-index.xml
https://dl.broadcom.com/<Download Token>/PROD/COMP/ESX_HOST/addon-main/vmw-depot-index.xml
https://dl.broadcom.com/<Download Token>/PROD/COMP/ESX_HOST/iovp-main/vmw-depot-index.xml
https://dl.broadcom.com/<Download Token>/PROD/COMP/ESX_HOST/vmtools-main/vmw-depot-index.xml

These four URLs cover the main ESXi depot, add-ons, I/O vendor program drivers, and VMware Tools respectively. You'll want all four to have complete access to updates.

üí° Tip: Save these URLs somewhere secure. Your token is essentially a password to your Broadcom downloads, so treat it accordingly.

Configure vCenter Lifecycle Manager

Now let's add these URLs to Lifecycle Manager so you can download updates directly from vCenter.

Step 1: Open Lifecycle Manager Settings

Log into vSphere Client as an administrator. Click the hamburger menu (three horizontal lines) in the top left corner and select Lifecycle Manager.

Step 2: Add the New Repository URLs

Navigate to the Settings tab, then click on Patch Setup. Click New to add a new download source.

Paste each of the four URLs you created earlier, adding them one at a time. Give each one a descriptive name so you can identify them later (e.g., "Broadcom ESXi Main", "Broadcom ESXi Add-ons", etc.).

Step 3: Enable the New Sources

After adding the URLs, check the Connectivity Status column. If any show as "Not Connected", click Enable to activate them.

You can also disable the old VMware URLs at this point since they're no longer functional. This keeps your configuration clean and avoids sync errors.

Step 4: Sync the Repository

Click Sync Updates to pull the latest package information from Broadcom's servers. This may take a few minutes depending on your connection speed.

Watch the Recent Tasks pane at the bottom of the screen. Once the sync tasks complete successfully, you should see the latest ESXi releases available in Lifecycle Manager.

Verify and Update

Once the sync is complete, head over to your cluster or host in Lifecycle Manager. You should now see the latest ESXi versions available for remediation. If you're running 8.0 Update 3b, for example, you should now see Update 3g (or whatever the current release is) as an available update.

From here, you can proceed with your normal ESXi update workflow through Lifecycle Manager ‚Äî create a baseline, check compliance, and remediate your hosts.

Wrapping Up

The Broadcom acquisition definitely added an extra step to the VMware update process, but once you've got your token configured, it works just as smoothly as before. The main thing to remember is that your token is account-specific, so if you're managing multiple environments under different Broadcom accounts, you'll need separate tokens for each.
