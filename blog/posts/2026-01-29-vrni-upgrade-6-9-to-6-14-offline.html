Steps to Upgrade vRealize Network Insight from 6.9.0 to 6.14.1 Using Single-Click Offline Upgrade

January 29, 2026 8 min read

I had a sudden request from a customer yesterday to upgrade their vRealize Network Insight deployment. They were running version 6.9.0, which is way out of support life. The plan was to upgrade to the latest release (6.14.1) before eventually moving to version 9.0.1.0.

This product has gone through several name changes over the years from vRealize Network Insight (vRNI) to Aria Operations for Networks, and now VCF Operations for Networks. Regardless of what it's called, the upgrade process remains similar.

Here's the complete walkthrough of how I performed this upgrade using the Single-Click Offline Upgrade method.

Pre-Requisites

Check Product Interoperability

As with any VMware upgrade, the first step is checking the VMware Product Interoperability Matrix. In this case, I verified compatibility with vCenter, ESXi, and Usage Meter. Everything checked out for upgrading to 6.14.1.

Determine the Upgrade Path

Here's where it gets interesting. You can't jump directly from 6.9.0 to 6.14.1. VMware doesn't support that upgrade path. After checking the documentation, I found the required upgrade path:

  • 6.9.0 ‚Üí 6.11.0 (first hop)
  • 6.11.0 ‚Üí 6.13.0 (second hop)
  • 6.13.0 ‚Üí 6.14.1 (final destination)

This means three separate upgrade cycles. I decided to upgrade to 6.11.0 first, let it run for a day to collect data and verify that the platform and collectors were working as intended, then continue with the remaining upgrades.

Download the ISO Files

Download all the required ISO files from the Broadcom Support Portal. You'll need the ISOs for versions 6.11.0, 6.13.0, and 6.14.1.

Backup Your Deployment

Before touching anything, back up your vRNI deployment. In my case, the customer didn't have backup configured, so I performed a cold snapshot of both the platform and collector VMs. Shut them down gracefully, take the snapshots, then power them back on.

Prepare the License

I generated a new license in advance, though the upgrade process didn't actually prompt for it. Still, it's good to have it ready just in case.

Check File System Usage

Make sure you have sufficient free space for the upgrade. Here are the requirements:

Platform (Node 1 only):

  • / ‚Äî 8GB free space
  • /var ‚Äî 40GB free space

Platform and Collector servers:

  • /tmp ‚Äî 1GB free space
  • /home ‚Äî 5GB free space

Change UI Timeout

The KB here suggests changing the UI timeout to 2 hours, but I set it to 4 hours. Better safe than sorry. You don't want the session timing out in the middle of an upgrade.

Verify Platform and Collector Status

Make sure both the Platform server and Collector server are online. If you took cold snapshots like I did, remember to power the servers back on before proceeding.

Record Current Values

Before upgrading, jot down these values from the UI for comparison after the upgrade:

  • Count of VMs
  • VMs where snapshot count > 0
  • Count of Firewall Rules
  • Count of Security Groups
  • Count of NSX Firewalls

You can find all this information by clicking through the UI. This helps verify that data wasn't lost during the upgrade.

Upgrade Process

I followed the Single-Click Offline Upgrade documentation from Broadcom. Here's how it went.

Step 1: Navigate to Offline Upgrade

Go to Settings ‚Üí Infrastructure and Support. Under the Product section, you'll see the Software Version, Online Update toggle, and Offline Update option. Click on Click Here beside Offline Upgrade.

Step 2: Upload the ISO

Browse and select the ISO file for version 6.11.0. The upload will take some time depending on your connection speed.

Wait for the upload to complete. You'll see a progress indicator.

Step 3: Start the Upgrade

Once the upload is complete, the status will show as available. Click View Details to start the upgrade process.

A popup window will appear showing the build number, information about what will happen during the upgrade, and a checkbox to proceed. Tick the checkbox and click Confirm.

Step 4: Pre-Check

A new popup shows the pre-check status. Review any warnings or issues, then click Install Now to proceed.

Step 5: Upgrade in Progress

And here's the main event, the upgrade begins. This is why we changed the UI timeout earlier. You don't want the upgrade to be interrupted, and I didn't want to find out what happens if the session times out mid-upgrade.

You can monitor the progress as the system upgrades various services.

Step 6: Upgrade Complete

Once the upgrade finishes, you'll see a completion message. Click Done.

Step 7: Verify the Upgrade

After clicking Done, you'll be redirected to the login page; now with the new interface.

Log in and verify the installed version. Compare the values you recorded earlier (VM count, firewall rules, security groups, etc.) to ensure everything came through correctly.

Rinse and Repeat

Since we can't jump directly to 6.14.1, you'll need to repeat the same process for each upgrade hop:

  • Upload the 6.13.0 ISO and upgrade
  • Upload the 6.14.1 ISO and upgrade

I recommend letting the system run for at least a few hours (ideally a day) between upgrades to verify data collection is working properly.

Notice that they have removed the Online Upgrade toggle.

Wrapping Up

Upgrading vRealize Network Insight across multiple versions takes patience, but the single-click offline method makes each individual upgrade straightforward. The key things to remember:

  • Check the upgrade path ‚Äî you may need multiple hops
  • Always backup before upgrading (snapshots work in a pinch)
  • Verify file system space requirements
  • Increase the UI timeout to avoid session interruptions
  • Record key metrics before and after for verification
