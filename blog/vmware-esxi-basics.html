VMware ESXi Basics: What I Wish I Knew Earlier

January 2, 2025 6 min read

VMware ESXi can seem intimidating at first—all those terms, the web interface, the command line. But once you understand the basics, it's actually pretty straightforward. Here's what I wish someone had explained to me when I started.

What is ESXi, Exactly?

ESXi is a bare-metal hypervisor. That means it installs directly on your server hardware (no Windows or Linux underneath) and lets you run multiple virtual machines on that single physical machine.

Think of it like this:

  • Physical server = One big house
  • ESXi = The apartment building manager
  • Virtual machines = Individual apartments

Each VM thinks it has its own hardware, but ESXi is actually sharing the physical resources among them.

ESXi vs vSphere vs vCenter

These terms confused me for months. Here's the simple breakdown:

  • ESXi — The hypervisor that runs on each physical server
  • vSphere — The overall product suite (ESXi + vCenter + other tools)
  • vCenter — Management server for controlling multiple ESXi hosts centrally

If you have one server, you just need ESXi. If you have multiple servers and want to manage them together, you add vCenter.

Getting ESXi

VMware offers a free version of ESXi for personal/lab use. It has some limitations (no vCenter integration, limited API access) but is perfectly fine for learning and small setups.

  1. Create a VMware account at vmware.com
  2. Search for "VMware vSphere Hypervisor" (the free version)
  3. Download the ISO
  4. Get your free license key (important—save this!)

Basic Installation

Installing ESXi is surprisingly simple:

  1. Flash the ISO to a USB drive
  2. Boot your server from the USB
  3. Follow the prompts (select disk, set root password)
  4. Wait for installation (usually 5-10 minutes)
  5. Note the IP address shown on the console
  6. Access the web interface at https://YOUR_IP

Tip: ESXi installs to a small partition. You can install it on a USB drive or SD card, leaving all your storage for VMs.

Key Concepts to Understand

Datastores

A datastore is where your VMs live—it's just storage that ESXi can see. It can be:

  • A local disk in the server
  • A SAN/NAS volume connected over the network
  • An NFS share

Name your datastores clearly (e.g., SSD-Local-500GB) so you know what you're working with.

Virtual Switches (vSwitches)

Virtual switches connect your VMs to the network. By default, ESXi creates one called vSwitch0 connected to your physical NIC.

You can create multiple vSwitches to:

  • Separate different types of traffic
  • Create isolated networks for testing
  • Connect to different physical NICs

Port Groups

Port groups are labels you assign to vSwitch connections. When creating a VM, you select which port group it connects to. Think of them as VLAN labels.

Common Tasks

Creating a VM

  1. In the web client, go to Virtual Machines
  2. Click Create / Register VM
  3. Choose "Create a new virtual machine"
  4. Name it and select the guest OS type
  5. Choose your datastore
  6. Configure CPU, RAM, disk, network
  7. Finish and power on

Uploading an ISO

To install an OS, you need to upload the ISO to a datastore:

  1. Go to Storage → select your datastore
  2. Click Datastore browser
  3. Create a folder called ISOs
  4. Click Upload and select your ISO file

Then when creating a VM, you can mount this ISO to the virtual CD/DVD drive.

Taking Snapshots

Snapshots save the state of a VM at a point in time. Super useful before making risky changes. 

# Via SSH (enable SSH first in host settings)
vim-cmd vmsvc/getallvms              # List all VMs
vim-cmd vmsvc/snapshot.create [vmid] "Before Update" "Pre-update snapshot"
vim-cmd vmsvc/snapshot.revert [vmid] [snapshotid]   # Revert if needed
vim-cmd vmsvc/snapshot.remove [vmid] [snapshotid]   # Remove when done

Warning: Don't leave snapshots running long-term. They grow over time and hurt performance. Use them for temporary protection, then delete.

Useful CLI Commands

SSH into your ESXi host (enable SSH in Host → Actions → Services) and try these: 

# System info
esxcli system version get
esxcli hardware cpu global get
esxcli hardware memory get

# Network
esxcli network nic list
esxcli network ip interface ipv4 get

# Storage
esxcli storage filesystem list

# VM operations
vim-cmd vmsvc/getallvms        # List VMs
vim-cmd vmsvc/power.on [vmid]  # Power on
vim-cmd vmsvc/power.off [vmid] # Power off (hard)
vim-cmd vmsvc/power.shutdown [vmid]  # Graceful shutdown

Common Gotchas

Things that tripped me up:

  • License expiration — The evaluation period is 60 days. Apply your free license key before it expires, or your VMs won't start.
  • SSH is disabled by default — Enable it in host services if you need command-line access.
  • Thick vs Thin provisioning — Thin saves space but can lead to overcommitment. Thick reserves the space upfront.
  • VMware Tools — Always install VMware Tools in your VMs for better performance and graceful shutdowns.
  • Hardware compatibility — Check the VMware HCL before buying hardware. Not everything works.

Next Steps

Once you're comfortable with the basics:

  • Set up automated backups (Veeam has a free version)
  • Learn about resource pools and reservations
  • Explore templates and cloning
  • Consider vCenter if managing multiple hosts

Wrapping Up

ESXi is powerful once you get past the initial learning curve. Start small—one host, a few VMs—and expand as you get comfortable. The skills transfer directly to enterprise environments, so it's worth learning properly.
