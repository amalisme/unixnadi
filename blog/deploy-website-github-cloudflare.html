Deploy a Static Website with GitHub + Cloudflare

January 5, 2025 8 min read

You don't need expensive hosting to put a website online. With GitHub Pages and Cloudflare, you can host a static website completely free‚Äîwith SSL, CDN, and excellent performance. Here's exactly how to do it.

What You'll Need

  • A GitHub account (free)
  • A Cloudflare account (free)
  • A domain name (optional, ~$12/year)
  • Your website files (HTML, CSS, JS)

Step 1: Create a GitHub Repository

First, let's get your files on GitHub.

  1. Go to github.com and sign in
  2. Click the + button ‚Üí New repository
  3. Name it something like my-website
  4. Make it Public (required for free GitHub Pages)
  5. Click Create repository

Step 2: Upload Your Files

You can either use Git from the command line or upload directly through the browser.

Option A: Using Git (Recommended)

# Navigate to your website folder
cd my-website

# Initialize git
git init

# Add all files
git add .

# Commit
git commit -m "Initial commit"

# Add your GitHub repo as remote
git remote add origin https://github.com/YOUR_USERNAME/my-website.git

# Push to GitHub
git branch -M main
git push -u origin main

Option B: Upload via Browser

  1. On your new repo page, click uploading an existing file
  2. Drag and drop your website files
  3. Click Commit changes

Step 3: Enable GitHub Pages

Now let's turn on hosting.

  1. Go to your repository on GitHub
  2. Click Settings (tab at the top)
  3. Scroll down to Pages in the left sidebar
  4. Under "Source", select Deploy from a branch
  5. Choose main branch and / (root) folder
  6. Click Save

Wait a minute or two, then your site will be live at: 

https://YOUR_USERNAME.github.io/my-website/

üéâ Congratulations! Your website is now online.

Step 4: Add a Custom Domain (Optional)

If you want your own domain (like example.com), keep reading. If the GitHub URL is fine for now, you can stop here.

4.1: Add Domain to Cloudflare

  1. Sign up at cloudflare.com
  2. Click Add a Site
  3. Enter your domain name
  4. Select the Free plan
  5. Cloudflare will scan your existing DNS records
  6. It will give you two nameservers‚Äîcopy these

4.2: Update Nameservers

Go to wherever you bought your domain (Namecheap, GoDaddy, etc.) and update the nameservers to the ones Cloudflare gave you. This usually takes 5-30 minutes to propagate.

4.3: Configure DNS Records

In Cloudflare, go to DNS and add these records: 

Type: CNAME
Name: @
Target: YOUR_USERNAME.github.io
Proxy: Yes (orange cloud)

Type: CNAME
Name: www
Target: YOUR_USERNAME.github.io
Proxy: Yes (orange cloud)

4.4: Tell GitHub About Your Domain

  1. Go back to your GitHub repo ‚Üí Settings ‚Üí Pages
  2. Under "Custom domain", enter www.yourdomain.com
  3. Click Save
  4. Wait for the DNS check to pass (may take a few minutes)
  5. Check Enforce HTTPS

4.5: Add CNAME File

Create a file called CNAME (no extension) in your repo root with just your domain: 

www.yourdomain.com

Commit and push this file.

Step 5: Configure Cloudflare Settings

A few tweaks to make everything run smoothly:

SSL/TLS Settings

  1. Go to SSL/TLS in Cloudflare
  2. Set encryption mode to Full

Page Rules (Optional)

To redirect non-www to www (or vice versa):

  1. Go to Rules ‚Üí Page Rules
  2. Add a rule: yourdomain.com/*
  3. Setting: Forwarding URL (301 Redirect)
  4. Destination: https://www.yourdomain.com/$1

Updating Your Site

Whenever you want to update your website: 

# Make your changes, then:
git add .
git commit -m "Update: description of changes"
git push

GitHub Pages will automatically rebuild your site in 1-2 minutes.

Troubleshooting

Common issues and fixes:

  • 404 errors ‚Äî Make sure you have an index.html file in the root
  • CSS not loading ‚Äî Check your file paths (use relative paths like css/style.css)
  • HTTPS not working ‚Äî Wait for DNS propagation, check Cloudflare SSL settings
  • Changes not showing ‚Äî Hard refresh (Ctrl+Shift+R) or clear cache

What You Get for Free

With this setup, you get:

  • ‚úÖ Free hosting (GitHub Pages)
  • ‚úÖ Free SSL certificate (HTTPS)
  • ‚úÖ Global CDN (Cloudflare)
  • ‚úÖ DDoS protection
  • ‚úÖ Fast page loads worldwide
  • ‚úÖ Easy deployment via Git

The only cost is your domain name (~$12/year), and even that's optional.

Wrapping Up
