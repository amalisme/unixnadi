Deploy a Static Website with GitHub + Cloudflare

December 5, 2025 8 min read

This is how I host UnixPulse — and several other static sites — completely free using GitHub and Cloudflare Pages. If you have a static website (pure HTML, CSS, JS), this setup gives you fast global delivery, automatic HTTPS, and continuous deployment from Git with zero monthly costs.

What You'll Need

  • A GitHub account (free)
  • A Cloudflare account (free)
  • Your static website files ready to deploy
  • A custom domain (optional, but recommended)

Step 1: Push Your Site to GitHub

Create a new GitHub repository and push your website files:

git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/yourusername/your-site.git
git push -u origin main

Keep your repository public or private — Cloudflare Pages works with both.

Step 2: Connect to Cloudflare Pages

Log into your Cloudflare dashboard and go to Workers & Pages → Create application → Pages → Connect to Git.

Authorize Cloudflare to access your GitHub account, then select your repository.

Step 3: Configure the Build

For a pure static site with no build tool, the configuration is minimal:

  • Framework preset: None
  • Build command: leave empty
  • Build output directory: / (or whichever subfolder contains your index.html)

Click Save and Deploy. Cloudflare will pull your files and deploy them to its edge network. Your site will be live at a *.pages.dev URL within a minute or two.

Step 4: Add a Custom Domain

In Cloudflare Pages, go to your project → Custom domains → Set up a custom domain. Enter your domain name.

If your domain is already on Cloudflare DNS (recommended), the CNAME record is added automatically. If it's on another registrar, Cloudflare will give you the DNS record to add manually.

HTTPS is automatic — Cloudflare provisions and renews the certificate for you.

Step 5: Automatic Deployments

Every time you push to your main branch, Cloudflare Pages automatically rebuilds and redeploys. For a static site, this takes about 30 seconds.

git add .
git commit -m "Update blog post"
git push

That's all it takes to publish a change. No FTP, no build servers, no manual uploads.

Why Not GitHub Pages?

GitHub Pages also works well for static sites, but Cloudflare Pages has better global performance (more edge locations), better analytics, and more flexible redirect/header rules. For a site where you're also managing DNS through Cloudflare, keeping everything in one place also simplifies the setup.
